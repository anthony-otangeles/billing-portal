Phase 2 — Settings & Team

Bell Medics manages its own people inside the Billing Portal: a Settings screen with the team roster and invite flow, the manager/rep role difference made visible, and the manager's per-rep workload strip on the Dashboard.

1 · Role switching — how you'll demo both sides

The role difference is the point of this phase, so the prototype needs a way to be both people. The avatar menu (top bar, every page) gains a "Sign in as" switcher with the two personas:

  • Angela Mensah — AR Rep (default, today's persona)
  • Derek Okafor — Billing Manager (the escalation target from the Agent Inbox)

Persisted like the practice switcher, so every page re-scopes to the chosen role. Built as a small shared component (same pattern as practice-switcher.js) so all six pages get it with one header change.

2 · New screen — Settings (Settings.dc.html)

Team tab MANAGER ONLY

Roster table of the real reps already in the claims data — Angela Mensah, Rob Tanaka, Priya Kapoor, Sam Osei + Derek — with role, assigned practices (IFC / Westbrook), status, last active, and live due-today load pulled from the shared claims store. Actions: Invite Rep (email + role + practice assignment → pending row until "accepted"), deactivate / reactivate, edit practice assignments. All changes persist and survive reload. As a rep, this tab shows a locked state explaining it's manager-only — the visible role boundary.

Company profile tab

Bell Medics name, primary contact, address, and a logo slot you can drop a real logo onto. Editable by the manager; read-only for reps.

Notifications tab

Per-user toggles, persisted per persona: SLA breaches, appeal outcomes, practice replies, weekly report ready. Both roles see this tab.
