9.1

Patient Statements — vendor handoff

effort S–M
  • Statements tab on Payments: every PATIENT-owned balance with its statement status, cycle (1st / 2nd / final notice), and dollar amount — the population Phase 8 made visible, now workable.
  • Vendor batch export: agent assembles the weekly statement file (print/mail + text-to-pay vendor, simulated); manager approves the batch; statuses flip to sent with the vendor reference on each ledger.
  • Returned-mail & payment-plan handling: exceptions come back as Agent Inbox decisions (bad address → practice queue; plan requested → approve terms).
9.2

Collections — governed handoff

effort S
  • Agent proposes, manager approves: after the final notice ages out (e.g. 120 days patient-owned, 3 statements, no contact), the agent proposes the account for collections — a decision with the full contact history as evidence.
  • Export + status: approved accounts land in an agency export file; balance status becomes UNCOLLECTIBLE-REQUESTED on the ledger, visible in the Register and the Practice window.
  • Practice veto: the practice sees proposed collections accounts in their window before export — they can pull any resident off the list. This is post-acute care; the practice owns that relationship.
