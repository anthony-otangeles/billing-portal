Phase 5 — Proactive Coverage Intelligence

The AR report is a rear-view mirror: nearly every denial in it traces to a data fact that was knowable before the visit. Two insights govern the design. First, 60% of preventable dollars come from just two failure modes (coding rules + COB). Second, most buckets are one root cause fanning out into dozens of denials — a payer rule, a taxonomy setup, a missing auth. So the system needs two kinds of memory: a per-patient coverage profile and a per-payer rulebook that turns every denial into a rule that prevents the next one.

2 · The concept — agent verifies, humans handle exceptions

Per your answers: the agent runs checks automatically on a pre-visit schedule, designed for the ideal data source (real-time 270/271 eligibility + payer portals), and people see only what the agent can't resolve. For every patient on the upcoming schedule, the agent verifies six things:

  • Eligibility — coverage active on the visit date, member ID valid, plan type correct.
  • Payer stack / COB — all payers known, primacy order confirmed, COB forms current.
  • Prior authorization — does this payer require auth for the planned service; is one on file.
  • Benefits — what's covered, visit limits used vs. remaining, deductible/copay state, exhaustion date.
  • Status flags — hospice election, capitation assignment, MA plan changes.
  • Provider enrollment — rendering provider credentialed + taxonomy correct with this payer.

Every check result lands on the patient's coverage profile — your "complete understanding of each patient from an insurance point of view." The end state you described: before a clinician sees the patient, the system already knows who pays, what they pay for, what they require, and how much benefit is left.

Verification run summary (verified clean / exceptions / re-checking), practice-wide flags (fix-once root causes like the Medicare taxonomy setup), and the exception queue — each row shows what the agent found, what it already did, and what it needs from a human. Clicking a row opens the full patient coverage profile: payer stack with primacy + status, six-check panel, benefits snapshot, and the agent's activity log. Populated with real cases from the AR report.
