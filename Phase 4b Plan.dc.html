Phase 4b — Roles & Portal Configuration

The governance layer: a definition of what each role can do, written down and visible in the Ops portal — so the permission boundaries the prototype enforces implicitly (Angela vs Derek vs Michael, tenant isolation) become explicit, reviewable artifacts. This is the last planned phase of the PRD.

1 · New screens (2)

Roles & Permissions Ops Roles.dc.html

A new System-group nav item. One card per role — Operations Admin, Practice Admin, Provider, Billing Manager (NEW), AR Rep (NEW) — with member count and scope statement. Selecting a role opens a permission matrix (capability × allow/deny) grouped by domain: Claims & AR, Team, Practices, Patient Data (PHI), Audit. The PHI row makes isolation explicit: AR Reps and Billing Managers see PHI only for practices their company serves; Ops sees none. Read-only matrix with an "edit requests are audited" affordance — role editing is a real-product feature we only need to gesture at.

Portal Configuration Ops Portal Config.dc.html

Per-portal feature switches, one column per portal (Provider, Scribe, Clerk, Billing — NEW): Agent Inbox autonomy level (draft-only / act-with-approval / act-and-report), SLA windows, appeal filing, weekly report cadence, 2FA enforcement. Billing column is live-wired: flipping "Agent autonomy" to draft-only visibly disables the approve buttons in the Bell Medics Agent Inbox (read from a shared config store). The other three portals are static columns for context.

2 · Small ripples

  • Ops nav gains Roles & Permissions and Portal Configuration under System.
  • Config changes write audit events (visible in the Audit Log immediately).
  • Agent Inbox (Bell Medics) reads the autonomy setting — a one-line banner explains when approvals are disabled by Operations.

3 · Out of scope

  • Editable permission matrices (real RBAC editing) — read-only with audited-edit affordance.
  • Custom roles; per-user permission overrides.
  • A second live billing-company portal (the multi-tenant demo) — still available as an optional phase after this.

4 · Deliverables & QA

  • Agent Inbox autonomy wiring + explanatory banner.
  • QA Checklist gains "NEW — Phase 4b" (~6 steps), including the cross-portal autonomy round trip.
